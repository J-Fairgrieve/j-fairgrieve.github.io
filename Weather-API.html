Weather Feature Analysis

Github: Click Here

This project makes use of two APIs:

  • OpenWeatherMap API
  • Google Maps API

About the Project

Please note, the data for this project was collected on 25/09/2022. There are three sections in total for this project:

  1. Using the OpenWeatherMap API
    In this section, a list of city names are curated using a randomly generated set of 1,500 co-ordinates. These cities are subsequently used for the API calls, generating the weather information for analysis throughout the rest of the project.
    Overall, the API retreived the weather information for 560 cities.
  2. Analysing the Extracted Data
    A quick analysis was taken using the weather features collated (Max Temperature, Humidity, Wind Speed and Cloudiness). Any outliers are identified and highlighted, but not removed in this case. The features were then compared against the Latitude of each city co-ordinate to identify any trend in the weather as you approach the equator.
  3. Using Google Maps API
    Finally, the Google Maps API was used to plot the humidity changes across the globe. After defining the ideal weather conditions for a holiday, map markers were then generated to show the nearest hotel in relation to the city co-ordinates.

Analysis

Outliers

The easiest way to identify outliers visually is by creating box plots:

The box plots instantly identify the outliers present in both the Humidity and Wind Speed features. From these graphs we can also see that CLoudiness has the highest variance and Wind Speed appears to be skewed across slower speeds.

Latitude vs. Temperature

Note, visualisations were produced to compare each weather feature against latitude and can be viewed in the code file on GitHub. However, only the Max Temperature appeared to have a correlation so I have only included those charts here.

A clear curve can be seen. The highest observed temperatures can be seen where the Latitude is close to the equator. The further north or south you travel, the lower the temperature will be.

Linear Regression

Similarly to the previous section, linear regression models have been created for every weather feature but only the Max Temperature findings will be discussed. The other charts have very low R Squared values so do not explain a high proportion of the variance. However, all models can be viewed in the code file.
In order to perform the linear regression models, the data has been split into Northern/Southern hemisphere DataFrames. In the sample cities, there were 386 cities in the northern hemisphere and 174 cities in the southern hemisphere.

The northern hemisphere regression model has an R Squared value of 0.697, demonstrating a strong correlation between the Latitude and the Max Temperature. Looking at the graph, the variance appears to be greater as the latutude decreases.

At first sight, the southern hemisphere model definitely displays a higher variance than the northern model. This is demonstrated further with the lower R Squared value of 0.550. This could be due to the smaller sample size of southern cities in this dataset.
